# Product Vision

## Purpose
Defines why SchoolClerk exists, who it serves, and the value it delivers.

## How To Use
- Update when product strategy or customer segment shifts.
- Use as reference for prioritization and tradeoffs.
- Keep statements measurable where possible.

## Template
## Vision Statement
SchoolClerk is the education operating system for multiple institution types, enabling each tenant to run operations digitally with a configurable, non-hardcoded platform.

## Problem Statement
Many institutions still rely on manual attendance, billing, fragmented student records, and rigid software that only fits one education model, causing errors, delays, and poor reporting.

## Target Institutions
- Pre-school
- Primary schools
- Secondary schools
- Colleges
- Polytechnic institutions
- Universities
- Training institutes
- Religious schools (Islamic/Bible schools)
- Education groups

## Value Proposition
- Unified system for student, finance, academics, communication, and operations workflows
- Tenant-based setup for each school with scalable SaaS delivery
- Configurable institution model with module enable/disable per tenant
- Flexible academic hierarchy across terms/semesters, classes/levels, and optional departments/programs
- Faster operations, fewer admin errors, better institutional visibility

## Success Metrics
- Onboarding time per school
- Tenant configuration completion time (institution type + modules + hierarchy)
- Monthly active staff per tenant
- Invoice collection rate
- Attendance and result publication timeliness
