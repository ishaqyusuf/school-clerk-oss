# AI Workflow

## Purpose
Defines the standard workflow for AI-assisted development in this repository.

## How To Use
- Follow this sequence for every feature, bug fix, and refactor.
- Keep documentation synchronized with code changes.
- Use ADRs and bug logs for important learnings.

## Template
## Standard Flow
1. Read `brain/BRAIN.md` and related domain docs before coding.
2. Confirm existing patterns in `brain/system/` and `brain/engineering/`.
3. Implement minimal, composable changes.
4. Update relevant docs in `brain/` with the code change.
5. Add ADR in `brain/decisions/` for major architectural decisions.
6. Add bug memory in `brain/bugs/` for root-cause learnings.
7. Move task state in `brain/tasks/`.

## Definition of Done
- Code implemented and validated.
- Docs updated.
- Task status updated.
- New decisions/bugs captured when applicable.
