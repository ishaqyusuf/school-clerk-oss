# Project Index

## Purpose
Map of key repository areas so contributors and AI can navigate the codebase quickly.

## How To Use
- Update when major folders, apps, or packages are added/removed.
- Keep descriptions short and factual.
- Link to deeper docs where relevant.

## Template
## Repository Root
- `apps/`: user-facing applications (web/mobile/admin as applicable)
- `packages/`: shared libraries and internal packages
- `brain/`: project knowledge base and AI collaboration docs

## Known Workspace Map
- `apps/web`: SaaS landing page (marketing/public website)
- `apps/dashboard`: SaaS application (authenticated product app)
- `apps/api`: API application surface
- `packages/db`: Prisma/PostgreSQL data layer package
- `packages/api`: tRPC API package
- `packages/ui`: shared UI components package

## Documentation Map
- `brain/system/`: architecture and system internals
- `brain/product/`: product strategy and roadmap
- `brain/engineering/`: coding standards and development practices
- `brain/database/`: schema, relationships, migrations
- `brain/api/`: endpoints, contracts, permissions
- `brain/tasks/`: backlog and execution tracking
- `brain/decisions/`: ADR records
- `brain/bugs/`: bug memory logs
