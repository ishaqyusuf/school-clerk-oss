# 23 — Payroll, Wages, And Staff Finance History

**What to build:** Salary structures, wages, allowances, deductions, advances, and one-off labor payments become manageable finance workflows, with staff finance history visible on staff profiles.

**Blocked by:** 21 — Money-Out And Needs-Funding Payables

**Status:** done

- [x] Staff can have salary structures that generate or guide recurring salary obligations.
- [x] Teaching and non-teaching staff use the same payroll structure while remaining filterable by category or role.
- [x] Wages support daily, hourly, or task-based payments.
- [x] Allowances, deductions, advances, bonuses, and one-off labor payments are represented clearly.
- [x] Non-staff casual workers can be saved as reusable payees instead of staff profiles.
- [x] Salary/wage payments normally use the Salary/Wages Account and show Needs Funding when underfunded.
- [x] Staff profiles show salary structure, wage/salary history, advances, deductions, unpaid balances, and payment receipts.
- [x] Tests cover recurring salary, wage payment, advances/deductions, one-off labor payee payment, staff profile history, and account balance effects.
