## Question

How should the review step replace tabs and cards with one scrollable sectioned table while preserving the current ready/matched/attention behavior?

## Context

The current review UI groups rows in `Ready`, `Match Found`, and `Needs attention` tabs and renders each row as a card. The user wants a single scrollable table-like surface with section headers, each followed by rows in that section.

## Resolve

- Define the review sections and ordering:
  - Needs attention
  - Match found
  - Ready/new rows
  - any imported/skipped/failure section after execution, if needed
- Decide section header contents:
  - title
  - count
  - checked count
  - brief reason/status
  - optional section-level check/uncheck controls
- Define row table columns:
  - checkbox
  - student name with subtitle/status details
  - match summary
  - selected action
  - more actions
- Decide whether classroom and gender appear in the student subtitle, as badges inside the name cell, or in compact secondary columns.
- Define empty section behavior without bringing back tab-like navigation.
- Define desktop and mobile behavior for a table that must not require horizontal scrolling for common cases.
- Preserve existing row grouping logic: rows with unresolved gender/classroom/action/candidate stay blocked; matched rows remain distinct from no-match ready rows.

## Expected Answer

A screen-level table specification with section names, column definitions, responsive behavior, and state mapping from current verification rows to new sections.

## Comments

> *This was generated by AI during Wayfinder issue commenting.*

Use one scrollable review body with sections in this order: `Needs attention`, `Match found`, then `Ready to import`. Each section should have a slim header with title, total count, checked count, and optional check/uncheck controls. Rows should use a table/grid structure with columns for checkbox, student/name details, match summary, action, and more actions. Put gender, classroom, line number, and blocker text inside the name subtitle/badge area so the table stays narrow and mobile-friendly.
