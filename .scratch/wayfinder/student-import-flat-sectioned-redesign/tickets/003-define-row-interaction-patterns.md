# Define Row Interaction Patterns

Labels: `wayfinder:prototype`
Status: Open

## Question

How should each row expose name editing, match selection, action selection, classroom/gender fixes, search promotion, and secondary commands inside the simplified sectioned table?

## Context

Existing row cards expose many controls directly. The requested table should keep the visible scan path simple: name badges/chips, closest match summary, action dropdown, and more-actions dropdown. Match details should open from the match column in a popover-like surface.

## Resolve

- Name cell:
  - display name as first name, surname, and other name tiny clickable badges/chips
  - preserve existing name-structure options and reset behavior
  - include name-structure editing in more actions as a secondary route
- Match column:
  - show closest selected/recommended match
  - show confidence percentage
  - show `+N more` when additional candidates exist
  - open an accessible detail surface to inspect candidates and choose one
- Action dropdown:
  - show current row action as the visible value
  - include valid actions only where possible
  - disable or explain candidate-dependent actions when no candidate is selected
- More actions:
  - update name structure
  - search/select existing student
  - reset row edits
  - skip where valid
  - import row only if single-row import remains in scope
- Fix controls:
  - decide where missing gender and classroom assignment live in the row
  - ensure rows with missing fixes remain visibly blocked
- Decide mobile behavior for popovers/menus, likely switching to a sheet or inline expansion when a popover would be cramped.

## Expected Answer

A row interaction contract that lists primary visible controls, secondary menu items, validation/disabled states, and the surface used for match candidate selection.

## Comments

> *This was generated by AI during Wayfinder issue commenting.*

Keep only the high-frequency row decisions visible: editable name chips in the name cell, closest match in the match cell, action dropdown, and more-actions menu. The match cell should open a candidate picker showing all matches, confidence, classroom, term/session status, and selected state. Move lower-frequency tools into the more menu: update name structure, search existing students, reset row edits, skip where valid, and single-row import if retained.
