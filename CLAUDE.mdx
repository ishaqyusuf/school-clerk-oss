Open-source multi-tenant school management SaaS. Built as a Turbo monorepo with Bun.

## Key Patterns

### Authentication
```ts
const cookie = await getAuthCookie()
// Returns: { schoolId, sessionId, termId, sessionTitle, termTitle, domain, auth: { userId, bearerToken } }
// File: apps/dashboard/src/actions/cookies/auth-cookie.ts
```

### Server Actions
- Location: `apps/dashboard/src/actions/`
- Use `prisma` from `@school-clerk/db` directly
- Always call `getAuthCookie()` to scope to current school/session/term

### tRPC
```ts
// Server prefetch (page.tsx):
batchPrefetch([trpc.router.procedure.queryOptions({})])

// Client query (component):
const trpc = useTRPC()
const { data } = useSuspenseQuery(trpc.router.procedure.queryOptions({}))

// Mutation:
const { mutate } = useMutation(trpc.router.procedure.mutationOptions({ onSuccess() {} }))
```

### UI Components
- Import: `@school-clerk/ui/[component]`
- Examples: `button`, `card`, `badge`, `input`, `label`, `select`, `checkbox`, `tabs`
- Custom: `@school-clerk/ui/custom/page-title`, `@school-clerk/ui/custom/icons`
- Styles: `@school-clerk/ui/globals.css` (OKLCH color tokens)
- `transpilePackages: ["@school-clerk/ui"]` required in next.config.ts

### Standard Page Template
```tsx
import { PageTitle } from "@school-clerk/ui/custom/page-title"
import { HydrateClient, batchPrefetch, trpc } from "@/trpc/server"
import { Suspense } from "react"
import { ErrorBoundary } from "next/dist/client/components/error-boundary"
import { ErrorFallback } from "@/components/error-fallback"

export default async function Page() {
  batchPrefetch([trpc.router.procedure.queryOptions({})])
  return (
    <HydrateClient>
      <PageTitle>Page Name</PageTitle>
      <ErrorBoundary errorComponent={ErrorFallback}>
        <Suspense fallback={<TableSkeleton />}>
          <DataComponent />
        </Suspense>
      </ErrorBoundary>
    </HydrateClient>
  )
}
```

## Database Schema Key Models

### Tenancy
```
SchoolProfile → Sessions → Terms
```

### Students
```
Students → StudentSessionForm (per session) → StudentTermForm (per term)
```

### Finance
```
Fees → FeeHistory (per term) → StudentFee (per student) → StudentPayment → WalletTransactions → Wallet
Billable → BillableHistory → Bills → BillPayment → WalletTransactions
Wallet { name, type, schoolProfileId, sessionTermId } — accounting stream
```

### Staff
```
StaffProfile → StaffTermProfile (per term) → Bills (salary/payroll)
```

### Attendance
```
ClassRoomAttendance (session) → StudentAttendance (per student)
```

## Feature Patterns

### Attendance System
- **Models**: `ClassRoomAttendance` (session header) + `StudentAttendance` (per-student record)
- **Router**: `attendance` — `getClassroomAttendance`, `takeAttendance`, `getStudentAttendanceHistory`
- **Classroom sheet**: "Attendance" tab → session list + "Take Attendance" secondary panel
- **Student sheet**: "Attendance" tab → history list + summary stats + attendance rate %
- **Hook**: `useClassroomParams` — tabs: `["overview", "students", "subjects", "attendance"]`; secondaryTabs include `"attendance-form"`

### Accounting Streams
- **Model**: `Wallet { name, type, schoolProfileId, sessionTermId }` — unique per [name + school + term + type]
- **Router**: `finance.getStreams({ filter: 'term' | 'session' })` — returns wallets with balance
- **Balance**: sum of `WalletTransactions.amount` filtered by `type` (transfer-out/debit = outgoing)
- **Fund Transfer**: creates two `WalletTransactions` (type: "transfer-out" / "transfer-in")
- **Predefined streams**: "Services" (bill), "Payroll" (bill), "Sales" (fee) — auto-created on first use
- **Component**: `AccountingStreams` — filter toggle (term/session), stream cards, create + transfer forms

### Finance — Service Payments
- **Model**: `Bills` where `staffTermProfileId = null` — non-staff expenses
- **Router**: `finance.getServicePayments`, `finance.createServicePayment`, `finance.payServiceBill`
- **Pay flow**: creates `WalletTransaction` (debit) → `BillInvoice` → `BillPayment` → updates `Bills.billPaymentId`
- **Component**: `ServicePayments` — split pending/paid, inline pay form, expense preset buttons

### Payroll
- **Model**: `Bills` where `staffTermProfileId != null` — linked to `StaffTermProfile`
- **Router**: `finance.getPayroll`, `finance.createStaffBill`, `finance.payStaffBill`, `finance.getStaff`
- **Create flow**: find/create `StaffTermProfile` for staff → create `Bills` linked to it + Payroll wallet
- **Component**: `Payroll` — summary stats, staff bill list (pending/paid), pay form, salary title presets

### Student Finance (enhanced)
- **Payment**: `finance.applyPayment` — supports payment method (stored in `StudentWalletTransactions.description`)
- **Purchase**: `finance.createStudentPurchase` — creates `StudentPurchase` + `StudentPayment` (type=PURCHASE)
- **History**: `finance.getStudentPayments` — full payment list with status
- **Reverse**: `finance.reverseStudentPayment` — cancels `WalletTransaction` + `StudentPayment`, restores `StudentFee.pendingAmount`
- **Component**: `StudentTransactionOverview` — Pay Fees / Bill / Purchase action buttons + history with reverse

### Student Payment Flow
```
User pays → applyPayment()
  → StudentWalletTransactions (credit, stores paymentMethod in description)
  → for each pending StudentFee:
      → WalletTransactions (on fee wallet)
      → StudentPayment (type=FEE, linked to fee + term form)
      → StudentFee.pendingAmount decremented
```
